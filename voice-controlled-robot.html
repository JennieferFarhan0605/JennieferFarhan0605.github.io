Engineering Thesis & IoT Integration

Voice Controlled Robot
(Wheelchair Implementation)

Tech Stack

  • Core: Arduino Uno (ATmega328)
  • Software: MIT App Inventor (Android)
  • Comms: HC-05 Bluetooth Module
  • Drive: L293D Motor Driver & Relays
  • Power: Buck Converter (DC/DC)

Executive Overview

This project was engineered to address a critical accessibility gap: minimizing human intervention for mobility devices. By integrating Android OS with embedded systems, I developed a hands-free navigation aid that processes acoustic input into mechanical motion.

The system allows users to control a robotic wheelchair using five distinct voice commands (Forward, Backward, Left, Right, Stop), translating digital signals into high-torque physical movement.

Technical Implementation

1. Power Engineering & Load Management

Porting the prototype to a real-world wheelchair presented a significant challenge: the system needed to withstand a load capacity of 80kg. Standard microcontroller outputs are insufficient for the 12V 60RPM DC Geared Motors required for this torque.

To solve this, I engineered a power stage using a Buck Converter (DC-to-DC). This Switched-Mode Power Supply (SMPS) steps down voltage while stepping up current, ensuring the motors receive sufficient drive without draining the battery inefficiently. This architecture restricted efficiency losses to just 5–15%, ensuring consistent performance under varying load conditions.

2. Hardware Interface & Logic

The Arduino Uno serves as the central processing unit. It interfaces with the L293D Motor Driver, which acts as a bridge between low-voltage logic (5V) and the high-power motor requirements.

For directional control, I implemented H-bridge logic. To execute a "Left" turn, the system sends a reverse DC supply to the left motor and a positive DC supply to the right motor, pivoting the chassis on its axis. High-current relays were integrated to safely isolate the microcontroller from the heavy load of the wheelchair motors.

3. Signal Processing & Communication

The control interface was built using MIT App Inventor, leveraging Google Speech Recognition to transcode voice input into text strings.

These strings are transmitted via the HC-05 Bluetooth Module to the Arduino. I programmed the microcontroller to initiate serial communication at a baud rate of 9600. The code continuously polls for incoming characters; for example, if the data stream contains “Forward,” the system triggers Pins 10 and 12 to HIGH, engaging the forward drive sequence.
