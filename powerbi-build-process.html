Project Overview

This page documents the complete technical build of the Healthcare Operations Dashboard in Microsoft Power BI Desktop. The dashboard connects seven Excel tables into a unified star schema, with a dedicated DAX measures library driving all 6 pages.

The build followed a deliberate sequence: data model first, measures second, visuals last. This order matters — visuals built before measures are correctly defined produce misleading results that are hard to diagnose later.

Data Model Decision

Daily_KPIs acts as the fact spine. All 6 detail tables connect to it via Date — many-to-one, both-direction cross-filtering. Department is a shared dimension across 4 tables.

Measures Decision

All 45+ DAX measures live in a single _Measures table, separated from source data. The underscore prefix sorts it to the top of the Data pane for easy access.

Physician Filter Decision

On Leave = N is applied as a permanent page-level filter on the Physician page — not inside every measure. This ensures leave days never inflate patient or revenue totals.

Phase 1 Connect & Prepare the Data
1
Open Power BI Desktop

Launch Power BI Desktop and close the splash screen. Confirm the three left-panel icons are visible: Report, Data, and Model. Start in Report view.

2
Import the Excel Dataset

Home → Get Data → Excel Workbook → navigate to Healthcare_Operations_Daily_Dataset.xlsx. In the Navigator, check all 7 sheets. Click Transform Data — not Load.

  • Daily_KPIs · Department_Daily · Physician_Daily · Patient_Outcomes_Daily
  • Financial_Daily · Equipment_Daily · Staffing_Daily
3
Fix Headers & Column Types in Power Query

Each table has a title banner row above the real headers. For every table: remove top row → promote headers → set column types.

  • Home → Remove Top Rows → 1, then Use First Row as Headers
  • Date columns → Date · Department, Day of Week, Quarter, Day Type, On Leave → Text
  • Week No. → Whole Number · All ($) columns → Fixed Decimal · All (%) columns → Decimal Number
  • When done: Home → Close & Apply
4
Define Relationships in Model View

Click the Model icon. Use Home → Manage Relationships → New to create 6 relationships. Detail table always on the From side, Daily_KPIs always on the To side.

  • From: Department_Daily[Date] → To: Daily_KPIs[Date]
  • From: Physician_Daily[Date] → To: Daily_KPIs[Date]
  • From: Patient_Outcomes_Daily[Date] → To: Daily_KPIs[Date]
  • From: Financial_Daily[Date] → To: Daily_KPIs[Date]
  • From: Equipment_Daily[Date] → To: Daily_KPIs[Date]
  • From: Staffing_Daily[Date] → To: Daily_KPIs[Date]
  • Each: Cardinality = Many to one · Make this relationship active ✓ · Tab + Enter to confirm
Phase 2 Build the Measures Library
5
Create the _Measures Table

Home → Enter Data. Name the table _Measures, add a Placeholder column with value 0. Click Load. All DAX measures go here — never inside source tables.

6
Write Core Executive Measures

Select _Measures → Table Tools → New Measure. Enter each formula and set format immediately after committing.

Total Admissions = SUM ( Daily_KPIs[Total Admissions] )
Total Revenue = SUM ( Financial_Daily[Total Revenue ($)] )
Total Costs = SUM ( Financial_Daily[Total Costs ($)] )
EBITDA = [Total Revenue] - [Total Costs]
EBITDA Margin % = DIVIDE ( [EBITDA], [Total Revenue], 0 )
Avg Bed Occupancy % = AVERAGE ( Daily_KPIs[Bed Occupancy (%)] )
Avg Patient Satisfaction = AVERAGE ( Daily_KPIs[Patient Satisfaction (/10)] )
Avg Mortality Rate % = DIVIDE ( SUM ( Daily_KPIs[Mortality] ), SUM ( Daily_KPIs[Total Admissions] ), 0 )
7
Write Department, Clinical & Financial Measures
Dept Total Revenue = SUM ( Department_Daily[Revenue ($)] )
Dept Total Admissions = SUM ( Department_Daily[Admissions] )
Dept EBITDA Margin % = DIVIDE ( SUM ( Department_Daily[Net Margin ($)] ), SUM ( Department_Daily[Revenue ($)] ), 0 )
Dept Avg LOS = AVERAGE ( Department_Daily[Avg LOS (days)] )
Overall Success Rate % = DIVIDE ( SUM ( Patient_Outcomes_Daily[Successful Outcomes] ), SUM ( Patient_Outcomes_Daily[Cases] ), 0 )
HAI Rate % = AVERAGE ( Patient_Outcomes_Daily[HAI Rate (%)] )
Avg Protocol Adherence % = AVERAGEX ( Patient_Outcomes_Daily, Patient_Outcomes_Daily[Protocol Adherence (%)] )
Avg Denial Rate % = AVERAGE ( Financial_Daily[Denial Rate (%)] )
8
Write Physician & Workforce Measures

Physician measures always filter On Leave = "N" to exclude leave days from calculations.

Physician Total Patients = CALCULATE ( SUM ( Physician_Daily[Patients Seen] ), Physician_Daily[On Leave] = "N" )
Physician Total Revenue = CALCULATE ( SUM ( Physician_Daily[Revenue ($)] ), Physician_Daily[On Leave] = "N" )
Revenue per Patient = DIVIDE ( [Physician Total Revenue], [Physician Total Patients], 0 )
Physician Avg Satisfaction = CALCULATE ( AVERAGE ( Physician_Daily[Satisfaction (/10)] ), Physician_Daily[On Leave] = "N" )
Avg Vacancy Rate % = AVERAGE ( Staffing_Daily[Vacancy Rate (%)] )
Avg Agency Staff per Day = AVERAGE ( Staffing_Daily[Agency Staff] )
Total Overtime Hours = SUM ( Staffing_Daily[Overtime Hours] )
Weekday Avg Headcount = CALCULATE ( AVERAGE ( Staffing_Daily[Total Staff] ), Staffing_Daily[Day Type] = "Weekday" )
Weekend Avg Headcount = CALCULATE ( AVERAGE ( Staffing_Daily[Total Staff] ), Staffing_Daily[Day Type] = "Weekend" )
Phase 3 Canvas Setup & Theme
9
Create & Name 6 Report Pages

Click + at the bottom to add pages. Right-click each tab to rename.

  • Page 1: Executive Summary · Page 2: Department Performance · Page 3: Clinical Quality
  • Page 4: Financial Performance · Page 5: Physician Productivity · Page 6: Workforce & Staffing
  • Each page: right-click canvas → Page settings → Canvas Settings → Custom → 1280 × 720
10
Import the Colour Theme

Save the JSON below as healthcare-theme.json then import via View → Themes → Browse for themes.

{ "name": "Healthcare Operations",
  "dataColors": ["#9D174D","#0F766E","#15803D","#B45309","#1D4ED8","#DC2626"],
  "background": "#F9F9F7", "foreground": "#1C1917", "tableAccent": "#9D174D" }
Phase 4 Build All 6 Pages
11
Page 1 — Executive Summary

6 KPI cards top row. Charts below: monthly admissions + revenue (line & stacked column, dual axis), revenue by department (horizontal bar), bed occupancy trend (line with 88% target), admissions by day of week (column).

  • Date hierarchy fix: New Column → Month Label = FORMAT(Daily_KPIs[Date], "MMM") + Month No = MONTH(Daily_KPIs[Date]) → Sort by Column
  • Day order fix: New Column → Day No = WEEKDAY(Daily_KPIs[Date], 2) → Sort Day of Week by Day No
  • Revenue line colour: #0F766E · Admissions bars: #9D174D · Occupancy line: #1D4ED8
  • Analytics pane → Constant Line at 0.88 on occupancy chart → label "Target" → red
12
Page 2 — Department Performance

Slicers: Date + Department dropdown. Charts: EBITDA margin by dept (horizontal bar, conditional formatting), admissions by dept (horizontal bar), avg LOS by dept (horizontal bar), revenue trend top 4 depts (line, filtered to Top N by Dept Total Revenue).

  • Conditional formatting rules (Number): <0.15 → #DC2626 · 0.15–0.22 → #B45309 · >0.22 → #15803D
  • LOS bars: #B45309 · Admissions bars: #9D174D
13
Page 3 — Clinical Quality

Slicers: Date + Procedure Category dropdown. 4 KPI cards. Charts: success rate by category (horizontal bar, conditional formatting), protocol adherence vs success rate (scatter), HAI rate monthly trend (line with 1% target), cases by category (horizontal bar).

  • Scatter fields: X = Avg Protocol Adherence % · Y = Overall Success Rate % · Values = Procedure Category · Size = Cases
  • HAI chart Y-axis: Minimum 0.005 · Maximum 0.015 · Constant line at 0.01
14
Page 4 — Financial Performance

Slicers: Date + Department dropdown. 4 KPI cards. Charts: revenue vs costs monthly (clustered column, green + red), revenue stream breakdown (donut, 3 stream measures), EBITDA margin ranking (horizontal bar, conditional formatting), cost composition (donut, 4 cost measures).

  • Revenue streams: Inpatient #9D174D · Outpatient #0F766E · Procedure #B45309
  • Cost composition: Staff #9D174D · Supplies #B45309 · Equipment #1D4ED8 · Overhead #0F766E
15
Page 5 — Physician Productivity

Slicers: Date + Physician dropdown. Permanent page filter: On Leave = N. 3 KPI cards. Charts: revenue by physician (horizontal bar, sorted desc), patients seen by physician (horizontal bar, teal). Full 12-row summary table with satisfaction column.

  • Filters pane → Filters on this page → On Leave → is N → Apply filter
  • Table columns: Physician · Department · Physician Total Patients · Physician Total Revenue · Revenue per Patient · Physician Avg Satisfaction
16
Page 6 — Workforce & Staffing

Slicers: Date + Department dropdown. 4 KPI cards. Charts: agency staff by dept (horizontal bar, amber), weekday vs weekend staffing (grouped horizontal bar), overtime monthly trend (line).

  • Month Label for Staffing_Daily: New Column → Month Label = FORMAT(Staffing_Daily[Date], "MMM")
  • Weekday bars: #9D174D · Weekend bars: #0F766E · Overtime line: #B45309
Phase 5 Sync, Validate & Publish
17
Sync Slicers Across Pages

Click the Date slicer on Page 1 → View ribbon → Sync Slicers. Tick both columns for all 6 pages. Sync the Department slicer across Pages 2, 4 and 6 only.

18
Validate FY 2024 Totals

Set date slicer to full year (1 Jan 2024 – 31 Dec 2024) on Executive Summary and verify:

  • Total Revenue → ~$102.8M · EBITDA → ~$18.5M (18.0% margin)
  • Total Admissions → ~21,473 · Avg Patient Satisfaction → ~8.10
  • Radiology EBITDA Margin → ~31.6% (highest) · ICU → ~10.7% (lowest)
19
Save & Export

File → Save As → Healthcare_Operations_Dashboard.pbix. To share without Power BI Service: File → Export → Export to PDF captures all 6 pages as static images.
